## 🧭 How I'm Approaching This

1. **One topic at a time** — I won't jump ahead until a step's core concept is clear, even if all problems aren't done.
2. **Understand before speed** — First solve using brute force, then optimize. Speed comes with repetition, not shortcuts.
3. **Daily log** — Maintain a simple log (date, problems solved, concept learned, doubts).
4. **Revision loop** — Use the "Revision" tab on the sheet weekly to prevent forgetting.
5. **No skipping "easy" steps** — Foundations (recursion, hashing, STL) directly decide how well Trees/Graphs/DP go later.
